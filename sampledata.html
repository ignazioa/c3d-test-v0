Sample C3D data files

This collection of files is a resource for programmers, developers and users who want to test their software applications for compatibility with the C3D standard. Each downloadable zip file contains a text file with additional information - the most recent additions to this collection are at the bottom of the page.

All software applications that claim C3D compatibility must be able to open and read the sample files that meet both the Original and Current C3D specifications as well as handle C3D files that contain errors in a sensible manner.

sample01.zip - C3D files in INT, REAL, PC, SGI and DEC formats

  • Original C3D specification - 762Kb download
  • This is a unique set of six C3D files created using ADTech AMASS software.  Each of the six files contain identical data and parameters in each of the C3D formats variants (DEC, PC, and SGI formats) in both floating-point and integer data types.  These files can be used to check that an application reads all of the C3D file types correctly.  Each file has been generated from the same original data.

sample02.zip - C3D file set for data format testing

  • Original C3D specification - 226Kb download
  • A file set that contains a set of six C3D files created using AMASS with Vicon "370" hardware - each file contains identical data and parameters in each of the C3D formats variants.  Note that these files locate the FORCE_PLATFORM:ORIGIN parameter incorrectly.

sample03.zip - C3D files containing human gait data

  • Current C3D specification - 516Kb download
  • Three sample C3D files containing human gait data generated on an Vicon "370" motion capture system.  These are; a normal adult gait trial, the same file after processing to add gait events, kinematics, and kinetics information.   An additional copy of the output file has been generated with a non-zero start time.

sample04.zip - C3D files containing human gait parameters

  • Current C3D specification - 6.4Mb download
  • Two files - bodybuilder.c3d is a large C3D file generated using Vicon "370" software and is stored in DEC floating-point format.  The second file, sub_labels.c3d, uses an interesting form of labeling to identify two subjects for the 3D point data. This is stored in DEC integer format and some of the calculated POINT variables show a loss of resolution as the data values approach the POINT:SCALE value.

sample05.zip - large C3D file containing human gait data

  • Current C3D specification - 4.8Mb download
  • A single, very large, C3D file containing human gait data generated using Vicon "370" software and stored in DEC floating-point format.

sample06.zip - C3D file with incorrect parameter names

  • C3D file with parameter name errors - 79Kb download
  • A sample C3D file from a Motion Analysis Corporation motion capture system that uses an old set of group and parameter names that deviate from the C3D standard. This file contains signed analog data and is stored in SGI integer format.

sample07.zip - C3D file with 16-bit analog data

  • Current C3D specification - 140Kb download
  • A sample C3D file from Vicon that contains 16-bit analog data - note the values for the parameter ANALOG:OFFSET.  Stored in PC floating point format.

sample08.zip - C3D parameter compatibility test suite

  • Original C3D specification - 540Kb download
  • A set of C3D test files in Intel/MS-DOS, INT format data generated by Motion Lab Systems.  These files can be used to check that your C3D compatible application reads the correct values from the C3D header in block 1 of the C3D file as required by the Label and Range specification (however, the files do not contain Label and Range data). All files contain identical 3D and parameter data but place the parameter and 3D data sections in different locations.
  • All C3D compliant applications should be able to read all of these files correctly.  Many commercial applications fail to read one or more of these sample files because they do not process the C3D file header pointers correctly.

sample09.zip - C3D files containing Vicon gait information

  • Current C3D specification - 608Kb download
  • A set of C3D files created by Vicon Workstation 4.4 with kinematic processing that has added additional gait parameters and kinematic variables to the C3D file.

sample10.zip - TYPE-2 and TYPE-4 force plate data

  • Original C3D specification - 111Kb download
  • These are two C3D files have been derived from a single source and contain identical force plate data. The file type-2.c3d describes the force plates as a TYPE-2 plate using only the major diagonal components of the force plate sensitivity matrix while type-4.c3d describes the force plate as a TYPE-4 plate using the entire sensitivity matrix.  A copy of the AMTI force plate sensitivity matrix information is included in a separate file together with an Excel spreadsheet that illustrates the output calculations using the entire calibration matrix.  These files can be used to verify or test application support for TYPE-4 force plates.

sample11.zip - C3D files with poor Force Plate data

  • Noisy Force Plate data - 277Kb download
  • Two sample C3D files from different laboratories using different 3D systems and different force platforms. Both C3D files contain unexpectedly noisy analog data in the force plate channels.

sample12.zip - A large 100Mb+ C3D file

  • Current C3D specification - 952Kb download
  • This is a single C3D file that contains over 100Mb of 3D data in one trajectory - a total of 18,178 frames at 60Hz. An excellent test file for C3D software applications to examine their ability to handle large C3D files.

sample13.zip - C3D files from PhoeniX Technologies

  • Current C3D specification - 435Kb download
  • Four sample C3D data files from PhoeniX Technologies - the first two files contain format errors that will cause some C3D applications to crash, while the second pair of files contain the same data with the formatting errors corrected.  This data is interesting in that it does not use +Z as the vertical axis and the two sets of files illustrate the subtle differences between good and bad C3D files.

sample14.zip - C3D file with analog data errors

  • This file contains Analog and 3D data errors - 116Kb download
  • A single C3D file that contains 3D and analog data that illustrates a loss of 3D data and analog data synchronization - in this example the analog data has been written to the file incorrectly as can be clearly seen when looking at the force plate Fz channels.

sample15.zip - C3D file with more than 127 3D points

  • Current C3D specification - 388Kb download
  • A C3D file that contains more than 127 3D points - this file uses unsigned integers within the C3D parameter section to store up to 255 3D points within the POINT:LABELS parameter - C3D applications that expect signed integers (particularly as array indexes) may not be able to read this data correctly.

sample16.zip - C3D file with invalid data points

  • This file contains invalid 3D data - 131Kb download
  • Three sample C3D files produced by a SIMI motion capture system. These file are in floating point IBMPC format and have their point residuals set to -1 indicating that the data points are invalid. As a result most C3D applications will report that these files do not contain any valid data points.

sample17.zip - C3D file with 128 analog data channels

  • Current C3D specification - 1.17Mb download
  • A sample C3D file from a Vicon system with 128 channels of analog data.

sample18.zip - C3D file with an invalid parameters

  • This file contains a corrupted parameter section - 193Kb download
  • This C3D file contains a corrupted parameter section and may cause some applications to crash if they assume that the parameter section is correctly formatted. The problem in this file is that there seems to be an "empty" group which has a name with length=9, but actually contains no data. This is the group which is reporting a zero offset of -1. Strictly speaking when all the groups/parameters are done, the next byte should have a zero value.

sample19.zip - C3D file with 34672 frames of data

  • Current C3D specification - 2.25Mb download
  • This C3D file contains 34672 frames of analog data (two channels) stored as a PC-Real formatted file. Note that this file will not be read correctly by any application that reads the C3D header or the parameters as 16-bit SIGNED values.

sample20.zip - C3D file with missing parameters

  • Missing parameter information - 317Kb download
  • A C3D file from PhaseSpace that contains a 3D data block but no parameters to describe the data. As a result most C3D applications will have trouble reading this file correctly because there is no information (parameters) about the data.

sample21.zip - C3D file with the wrong parameter values.

  • Bad parameter information - 93Kb download.
  • The actual 3D data frame rate is 200 and the analog sample rate is 1000Hz. However the parameters report (incorrectly) that the POINT:RATE is 60 and the ANALOG:RATE is 1020 - both are incorrect. As a result most C3D applications will not read this file correctly.

sample22.zip - C3D file from a robotic exoskeleton

  • Current C3D specification - 215Kb download
  • A C3D file generated by the BKINtechnologies Dexterit-E software which is controlling a 2D KINARM robotic exoskeleton for the arm. The data collected from the exoskeleton is stored using the C3D format.
  • The file contains EVENT parameters that resemble, but are not identical to, the Vicon method of parameter based EVENT storage.

sample23.zip - C3D file containing ANALYSIS parameters

  • Current C3D specification - 898Kb download
  • A C3D file of human gait with force plate and EMG data that contains ANALYSIS parameters values for stride length, cadence, support times etc, calculated by the Vicon Clinical Gait Software.
  • The file also contains EMG parameters used by the Motion Lab Systems to control their EMG Graphing and Analysis software.

sample24.zip - C3D file from Innovative Sports Training.

  • Missing parameter information - 234Kb download.
  • Sample C3D file data from an Innovative Sports Training MotionMonitor 3D system with blank data labels which will cause most C3D applications to fail has there is no way to distinguish one 3D point from another.

sample25.zip - C3D files from a Vicon Nexus system

  • Current C3D specification - 6.1Mb download.
  • Five sample C3D files from Vicon that contain prescaled force plate data. The 01 and 02 trials feature POINT and ANALOG data from a subject dancing on the plate, while the 03 and 04 trials are simple walks over the force plates producing recognizable force plate gait data.
  • Although the C3D file is correctly formatted, the poor choice of the analog scaling factors (GEN_SCALE = 1.0 and SCALE = 1.0) has the potential to cause data corruption when the analog data in the file is converted to INTEGER format.

sample 26.zip - C3D files from Qualisys.

  • Current C3D specification - 3.2Mb download.
  • A collection of C3D files provided by Qualisys illustrating human gait.

sample27.zip - C3D file with Kyowa Dengyo data.

  • Current C3D specification - 17Kb download.
  • A sample C3D file containing analog data from three Kyowa Dengyo force plates. The data from these plates is stored using the TYPE 3 (8 channel KISTLER) force plate description and can be processed in the same way as standard TYPE-3 data.

sample28.zip - C3D files with TYPE-1 force plate data.

  • Bad parameter information - 234Kb download.
  • Two sample C3D files from a BTS system using their Biomech 2.5 software. These files are interesting because they contain TYPE-1 force plate data but since the files are missing several important parameters (e.g. POINT:RATE, POINT:DATA_START etc.) and contain duplicate POINT labels some applications may fail to read the data correctly.
  • A third C3D file (type1.c3d) has been created that corrects the BTS formatting errors to provide sample TYPE-1 force plate data that should be readable by an C3D complient application.

sample29.zip - C3D files from NaturalPoint.

  • Current C3D specification - 4.08Mb download.
  • Two sample C3D files generated by NaturalPoint processed with their ARENA motion capture software. The files contain point data only.

sample30.zip - C3D files from Biogesta.

  • Current C3D specification - 426kb download.
  • Three sample C3D files generated by Biogesta processed with their Saga-3rt software. The files contain both 3D point and analog data.

sample31.zip - C3D files with more than 65535 frames.

  • Current C3D specification - 162Mb download.
  • Two sample C3D files generated by a Vicon Nexus system that each contain both 3D point and analog data. Each file contains more than 65535 frames of data - as a result they can only be opened by applications that support the use of the TRIAL group to store frame numbers.

sample32.zip - C3D file with corrupt EMG data.

  • Current C3D specification - 370kb download.
  • This file demonstrates a problem with importing data from a Cometa/Aurion ZeroWire EMG system into Vicon via USB, causing corrupt data that is not properly time frame aligned with the master frame in Nexus. The data appears to be noisy but spectrum analysis shows that the import process has corrupted the EMG data.

sample33.zip - C3D file with large parameter block.

  • Current C3D specification - 668kb download.
  • This static C3D data file contains a large parameter block resulting in a DATA_START value of 188 which exceeds the original C3D specification limit of +127 max. It may crash applications that interprete the DATA_START parameter as signed instead of unsigned.

sample34.zip - C3D files from Xsens.

  • Current C3D specification - 1.57Mb download.
  • Three sample C3D data files exported from the MEMS based Xsens motion capture system containing 3D data only.

sample35.zip - C3D files from Mega Electronics.

  • Current C3D specification - 267kb download.
  • A sample C3D file containing EMG data created by the Mega Electronics product MegaWin, containing an isokinetic measurement with two channels of EMG, knee angle and torque.
